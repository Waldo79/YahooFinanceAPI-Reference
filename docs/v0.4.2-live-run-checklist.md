# v0.4.2 Live-Run Validation Checklist

## Purpose

Use one fresh real-world Quote capture to verify v0.4.2 before making additional capture-utility changes.

This step is observational. Do not edit raw responses, metadata sidecars, normalized output, or the run manifest after capture.

## Run from the repository root

### 1. Validate the symbol table

```text
py tools\capture-utility\yahoo_capture.py --dry-run
```

Confirm that the enabled symbols and request order are correct.

### 2. Run the capture

```text
py tools\capture-utility\yahoo_capture.py
```

Record the new run-folder name created under:

```text
captures\local
```

### 3. Validate the completed run

Replace `<run-folder>` with the folder created in step 2:

```text
py tools\capture-utility\yahoo_capture.py --validate-run captures\local\<run-folder>
```

## Review points

Confirm all of the following:

- the run folder was created under repository-root `captures/local/`;
- progress lines appeared in sequence for every enabled symbol;
- the run manifest does not expose a Windows username or full local directory path;
- repository inputs are recorded as repository-relative paths;
- external inputs, if any, are recorded by filename only;
- raw SHA-256 and response-byte-count checks pass;
- manifest entries and metadata sidecars agree;
- no unredacted crumb, cookie, or authorization value is reported;
- no unexpected missing, duplicate, unsafe, or unreferenced file is reported;
- `run-validation.json` and `run-validation.txt` are created; and
- final validation status is `PASS` or `PASS_WITH_WARNINGS`.

If validation status is `FAIL`, preserve the run unchanged and record the issue codes before making any correction.

## Report only the review summary

Record:

- run-folder name;
- capture exit code;
- validation exit code;
- validation status;
- error, warning, and information counts;
- validation issue codes;
- known JSON-path count; and
- unmapped JSON-path count.

Do not include cookie values, crumb values, authorization values, or complete local paths in a public report.

## Evidence handling

Keep `captures/local/` local unless a specific run has been reviewed for privacy and intentionally selected for publication. Validation reports are derived review artifacts and do not replace the original raw evidence or manifest.
