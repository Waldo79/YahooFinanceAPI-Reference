# Anonymous Yahoo Session Support — v0.4.1

## Reason for this release

The first live v0.4.0 capture on 2026-07-11 successfully executed and wrote a complete run folder, but all 16 Quote requests returned HTTP 401 Unauthorized. This established that a bare Quote request was no longer sufficient for that test environment.

## Implemented session workflow

```text
Create in-memory cookie jar
  → request anonymous Yahoo cookie
  → request temporary crumb
  → add crumb to Quote URL
  → send sequential symbol request
  → refresh session once after HTTP 401/403
```

## Strategies

The utility first uses:

- cookie bootstrap: `https://fc.yahoo.com`
- crumb endpoint: `https://query1.finance.yahoo.com/v1/test/getcrumb`

If that does not produce a usable crumb, it tries:

- cookie bootstrap: a normal Yahoo Finance quote page
- crumb endpoint: `https://query2.finance.yahoo.com/v1/test/getcrumb`

These are observed, unofficial Yahoo interfaces and may change.

## Evidence and privacy rules

The temporary cookie and crumb are operational session values, not evidence fields. They remain in memory and are never written to disk.

Public output records only:

- authentication mode;
- non-sensitive strategy name;
- whether a crumb was present;
- cookie count;
- refresh count;
- redacted request URL; and
- non-sensitive setup errors, if any.

## HTTP 401 and 403 handling

The utility performs at most one forced anonymous-session refresh for the entire run. This prevents a permanent authorization failure from causing repeated cookie and crumb requests for every symbol.

If the refreshed request still fails, the final Yahoo response is preserved and classified normally as an HTTP error.

## Validation

The offline suite verifies:

- cookie and crumb setup order;
- percent-encoded crumb insertion;
- crumb redaction from manifest and metadata;
- one session refresh after HTTP 401;
- success after a refreshed crumb;
- Query2 fallback when Query1 returns an invalid crumb; and
- all v0.4.0 capture, normalization, retry, and integrity behavior.

Live Yahoo behavior cannot be confirmed from the build environment and must be tested from an internet-connected user system.
