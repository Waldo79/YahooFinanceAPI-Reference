# Zero-Pause Capture Baseline — v0.4.3

## Reason for this release

The v0.4.2 capture utility used a fixed 1,000-millisecond pause after each completed symbol. That setting was deliberately conservative, but it added avoidable time to normal sequential Quote captures.

Repeated live stopwatch tests on July 16, 2026 showed that the tested 16-symbol workflow completed successfully without the added delay. v0.4.3 therefore makes 0 milliseconds the normal inter-symbol pause while retaining every existing retry, timeout, authentication-refresh, evidence, and validation safeguard.

## Live timing evidence

The same 16-symbol capture workflow was tested three times:

| Pause setting | Elapsed stopwatch time | Result |
|---:|---:|---|
| 0 ms | 3.82 seconds | Successful |
| 25 ms | 4.31 seconds | Successful |
| 0 ms repeated | 3.65 seconds | Successful |

These observations establish the project default for the tested workflow. They are not a promise that Yahoo will never rate-limit or change its behavior.

## Related completed verification

The separate row-by-row app verification completed before this upload package was built used the Global 130 and U.S. 130 universes. The two lists contained 260 displayed assignments with 23 overlaps, producing 237 unique symbols. The final corrected symbols were `CWEN` and `INGA.AS`, and the 69-column output layout was accepted as the verification layout.

That verification supports the release record but does not change the capture utility's existing limit of 30 enabled symbols per run.

## New normal default

The utility now defines:

```text
DEFAULT_PAUSE_MS = 0
```

Running the normal command:

```text
py tools\capture-utility\yahoo_capture.py
```

sends the next sequential symbol request as soon as the prior symbol has been fully processed and its evidence has been written. The utility still sends only one symbol per Quote request and does not introduce parallel requests.

## Explicit pacing override

`--pause-ms` remains available for users who want a fixed delay:

```text
py tools\capture-utility\yahoo_capture.py --pause-ms 1000
```

Any nonnegative integer is accepted. A negative value remains an input error.

## Safeguards that did not change

The zero-pause default affects only the normal delay between completed symbols. It does not change:

- up to three attempts per symbol by default;
- retry delays of two and five seconds by default;
- retry handling for network failures and HTTP 429, 500, 502, 503, and 504;
- the 30-second default per-attempt timeout;
- the one-time anonymous-session refresh after HTTP 401 or 403;
- in-memory cookie-and-crumb handling and secret redaction;
- byte-for-byte raw-response preservation;
- SHA-256, metadata sidecars, normalized output, and run manifests;
- repository-root output and portable path references; or
- completed-run validation and privacy scanning.

## Compatibility

- Python 3.10 or newer
- Python standard library only for capture and validation
- `pytest` optional for development tests
- Existing capture folders remain valid and can still be checked with `--validate-run`
- Validation schema version remains v0.4.2 because the validation report structure did not change

## Validation

The v0.4.3 source package is checked with:

```text
py -m pytest -q
py -m py_compile tools\capture-utility\yahoo_capture.py
```

The test suite includes direct checks that the parser uses 0 ms by default and preserves an explicit `--pause-ms 25` override.

## Evidence rule

The faster normal pacing does not alter the evidence standard. Raw Yahoo response bodies remain unchanged, and capture context remains in filenames, metadata sidecars, normalized output, and the run manifest.
