# First Working Capture Utility — v0.4.0

## Purpose

v0.4.0 implements the first executable portion of the v0.3.9 capture-format and normalized-output specifications.

The release is intentionally limited to the Yahoo Finance Quote endpoint so the evidence model can be validated before additional endpoint families are added.

## Implemented workflow

```text
Editable symbol table
  → one Quote request per enabled row
  → unchanged response body
  → SHA-256 integrity value
  → metadata sidecar
  → normalized text
  → run manifest
```

## Input

`tools/capture-utility/symbols.csv` contains the 16 representative symbols across stocks, ETFs, closed-end funds, mutual funds, futures, indexes, cryptocurrency, and foreign exchange.

The utility preserves table order, skips disabled rows, rejects full URLs entered as symbols, and enforces the project maximum of 30 enabled rows.

## Evidence preservation

The body received from Yahoo is written byte-for-byte. It is not pretty-printed, sorted, decoded and re-encoded, or given a non-JSON text header.

Each received body has:

- a UTC-based filename;
- a SHA-256 digest;
- an associated metadata sidecar;
- an entry in the run manifest; and
- normalized text when the body is valid JSON.

## Result classifications

The implementation distinguishes:

- `SUCCESS_RESULT_RETURNED`
- `SUCCESS_EMPTY_RESULT`
- `REQUESTED_SYMBOL_MISSING_FROM_RESULT`
- `HTTP_ERROR`
- `RATE_LIMIT_OR_THROTTLE`
- `NETWORK_ERROR`
- `PARSE_ERROR`

An omitted requested symbol is not treated as an explicit JSON null.

## Retry policy

Retries are conservative and configurable. By default the utility makes at most three attempts and waits two seconds and then five seconds between retryable attempts.

Only network failures and HTTP 429, 500, 502, 503, and 504 responses are retried. The utility does not attempt to evade access controls or throttling.

## Normalized output

Valid JSON is flattened with full array indexes preserved. Known paths are ordered by their row order in `data/master_field_database.csv`; previously unmapped paths follow in deterministic JSON-path order.

Raw epoch values remain present. Recognized timestamp fields also receive a decoded UTC value.

## Validation

The offline test suite verifies:

- symbol-table parsing and URL rejection;
- array, null, empty-array, and empty-object flattening;
- byte-for-byte raw preservation and SHA-256 integrity;
- omitted-symbol classification; and
- retry behavior after HTTP 429.

## Deferred scope

Chart, QuoteSummary, Search, Screener, Options, scheduling, automated comparison, CSV generation, and workbook generation remain future releases.
