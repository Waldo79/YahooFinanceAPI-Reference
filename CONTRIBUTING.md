# Contributing

Contributions are welcome, especially raw Yahoo JSON captures, endpoint behavior notes, field descriptions, security-type comparisons, marketState observations, validation rules, and CSV mappings.

## Capture Quality

Good captures should include capture timestamp, symbol, endpoint, raw JSON response, observed `quoteType`, observed `marketState` when available, and notes about market session or unusual behavior.

## Parser Rule

Never rely on JSON field order. Yahoo may reorder fields between captures.
